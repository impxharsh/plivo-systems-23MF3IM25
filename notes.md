The architecture relies on a hybrid approach combining Forward Error Correction (FEC) and an Automatic Repeat reQuest (ARQ) feedback loop to neutralize network volatility. To combat isolated packet drops with zero round-trip delay, the sender computes a bitwise XOR across every pair of outgoing packets and transmits a third redundant block. If a single frame is lost, the receiver instantly reconstructs it using the surviving frame and the XOR parity block. To handle severe burst losses that wipe out entire FEC groups, the receiver detects sequence gaps and instantly fires NACKs back to the sender to request specific missing frames. A hash map in the receiver acts as a jitter buffer, naturally sorting out-of-order packets and safely discarding duplicates by overwriting identical sequence keys. 

**The delay_ms you should grade at is 110ms.** While the system successfully achieved 0.00% misses at 50ms on Profile A, extreme burst losses in Profile B required this higher threshold to guarantee absolute stability across unknown network conditions. The system will break if the network's Round Trip Time (RTT) plus the processing jitter exceeds the 110ms deadline. Specifically, if a massive burst loss forces a NACK, and the combined delay of the NACK reaching the sender and the retransmission crossing the relay is greater than 110ms, the recovered packet will miss the playout clock. Additionally, severe OS scheduler delays can trigger a false cascade of NACKs, which artificially shifts the relay's random number generator into a harsher drop pattern and risks breaching the 2.0x bandwidth cap.
